# PRD: Castone - Puerto Rico AI Battle Platform

## 1. 개요 (Overview)
Castone은 고전 보드게임 '푸에르토리코'를 기반으로 한 **실시간 다중 접속 AI 배틀 플랫폼**입니다. 유저들은 방(Room)을 생성하여 다른 유저 및 고도화된 AI(RL/MCTS)와 대결할 수 있으며, 모든 게임 데이터는 향후 모델 개선을 위해 정교하게 로깅됩니다.

## 2. 목표 (Goals)
- **실시간성**: WebSocket을 통한 지연 없는 상태 동기화.
- **확장성**: 수백 개의 게임 방을 동시에 관리할 수 있는 아키텍처.
- **AI 통합**: 다양한 수준의 AI(PPO, MCTS)를 유연하게 배치하고 대결.
- **데이터 기반**: 모든 게임 세션을 학습 데이터로 변환하는 MLOps 파이프라인 구축.

## 3. 핵심 기능 (Key Features)
### 3.1. 로비 및 방 관리 (Lobby & Room Management)
- 유저는 방을 생성하거나 목록에서 선택하여 입장할 수 있습니다.
- 방장은 빈 자리에 봇(Bot)을 추가하거나 시작할 수 있습니다.
- 게임 상태는 Redis 전역 세션으로 관리되어 서버 재시작 시에도 복구 가능해야 합니다.

### 3.2. 실시간 게임 플레이 (Real-time Gameplay)
- WebSocket 채널을 통해 액션 발생 시 모든 참여자에게 즉시 브로드캐스팅됩니다.
- 턴 제한 시간 및 자동 패스 로직이 서버 측에서 강제됩니다.

### 3.3. 유니버설 에이전트 인터페이스 (Universal Agent Interface)
- PPO(신경망), MCTS(탐색) 등 다양한 엔진이 동일한 `game_context`를 통해 의존성 없이 액션을 반환합니다.
- 봇의 행동은 인간 유저의 UX를 고려하여 인위적인 지연(2초 등) 후 실행됩니다.

### 3.4. MLOps 데이터 수집 (Data Collection)
- 매 턴의 `State`, `Action`, `Reward`가 기록됩니다.
- 게임 종료 시 전체 로그가 데이터 서버로 동영상/로그 형태로 아카이빙됩니다.

## 4. 비기능 요구사항 (Non-functional Requirements)
- **낮은 지연시간**: 봇 추론은 서버 로컬(ONNX/Torch CPU)에서 수행하여 네트워크 홉을 최소화합니다.
- **안정성**: PostgreSQL을 통해 유저 전적 및 모델 버전 이력을 관리합니다.
- **동시성**: Redis Pub/Sub을 사용하여 분산 환경에서도 동일한 방 유저 간 메시지 정합성을 유지합니다.

## 5. 성공 지표 (Success Metrics)
- 유저 액션 후 화면 반영까지의 지연 시간 < 100ms (네트워크 제외).
- 봇 추론 시 CPU 점유율 안정성 유지.
- 게임 로그의 결손율 0%.
