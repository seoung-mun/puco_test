📅 [Day 1: 금요일] 프론트엔드 시각화 및 인터랙션 완성
목표: 백엔드 없이도 모든 게임 상황을 시각화하고, 유저의 클릭을 '액션 인덱스'로 변환할 수 있는 완벽한 UI 구현.

[F1] 테마 및 레이아웃 프레임워크:

[ ] 보드게임 아레나(image_0.png) 스타일의 나무 바닥 배경 및 전체 그리드(상단 바, 중앙 메인보드, 우측 플레이어 패널, 하단 개인보드) 레이아웃 구축.

[ ] isMyTurn 상태에 따른 전체 화면 조작 잠금(Overlay) 및 알림 UI 구현.

[F2] 공용 자원 및 역할 컴포넌트:

[ ] 6종 역할 카드(선택 시 Grayscale), 3척의 수송선(상품 적재 시각화), 상점(4칸 슬롯), 공용 은행/이주민 공급처 UI 완성.

[F3] 개인 보드 및 그리드 시스템:

[ ] 12칸 농장/채석장 및 12칸 건물 그리드 구현.

[ ] [핵심] 200차원 action_mask를 수신하여, 클릭 가능한 타일/건물/버튼만 하이라이트하고 클릭 시 action_index를 캡처하는 로직 완성.

[F4] 상태 관리 및 Mock 데이터 연동:

[ ] React Context/Zustand 기반의 GameState 구조 설계 및 복잡한 observation 배열을 UI 객체로 변환하는 변환기(Mapper) 구현.

📅 [Day 2: 토요일] 통합 및 보안 레이어 구축 (Full-Stack Integration)
목표: Mock을 제거하고 실제 데이터 흐름을 연결하며, 보안 허점을 원천 봉쇄.

[I1] 실제 인증(Auth) 시스템 이식:

[ ] /mock-login 완전 삭제 및 Google OAuth 2.0 (/login/google) 연동.

[ ] 모든 API 및 WebSocket 연결 시 JWT 검증 미들웨어 적용. (RBAC: 관리자/일반유저 권한 분리 포함)

[I2] DB 스키마 업데이트 및 정합성:

[ ] Alembic을 통한 users 테이블 확장(email, elo_rating, role) 및 인덱스 최적화.

[ ] 게임 종료 시 전적 및 ELO 점수를 정산하여 DB에 자동 업데이트하는 서비스 로직 구현.

[I3] 실시간 양방향 통신 연결:

[ ] Redis Pub/Sub을 활용한 WebSocket STATE_UPDATE 브로드캐스트 엔진 완성.

[ ] 프론트엔드 useGameWebSocket 훅을 실제 백엔드 엔드포인트와 연결하여 실시간 리렌더링 검증.

[I4] 액션 파이프라인 연동:

[ ] 프론트엔드의 클릭(action_index)이 백엔드 엔진(env.step)을 거쳐 다시 프론트엔드로 돌아오는 전체 루프(Loop) 통합 테스트.

목표: AI 대전 기능을 활성화하고, 안정성을 검증한 뒤 실제 서버 환경으로 패키징.
