## 현재 검사에서 확인한 사실

### 1. 엔진은 "순차 커서" 모델이다

- 파일: `PuCo_RL/env/engine.py`
- `_init_mayor_placement()`가 시작 시 `recall_all_colonists()`를 호출하고 `mayor_placement_idx = 0`에서 시작한다.
- `_skip_empty_mayor_slots()`가 빈 슬롯을 건너뛰며 커서를 진행시킨다.
- `action_mayor_place(player_idx, amount)`는 현재 커서 슬롯 하나에 대해서만 `amount`를 받는다.
- 실제 의미:
  - `0~11`: island slot
  - `12~23`: city slot
  - 행동값 자체는 슬롯 지정이 아니라 `0~3명 배치`이다.

### 2. Env mask도 "현재 커서 슬롯에 몇 명 놓을 수 있는가" 모델이다

- 파일: `PuCo_RL/env/pr_env.py`
- Mayor action은 `69~72`만 처리한다.
- `69`는 현재 슬롯에 `0명`, `70`은 `1명`, `71`은 `2명`, `72`는 `3명`을 뜻한다.
- 즉 Mayor에서 action index는 슬롯 주소가 아니라 "현재 슬롯에서의 배치 수량"이다.

### 3. Serializer는 엔진 커서를 그대로 외부에 노출한다

- 파일: `backend/app/services/state_serializer.py`
- `meta.mayor_slot_idx = game.mayor_placement_idx`
- `meta.mayor_can_skip = action_mask[69]`
- 따라서 백엔드가 프론트에 알려주는 Mayor 진실 소스는 이미 "현재 커서 기반"이다.

### 4. 프론트와 action translator는 "슬롯 직접 지정" 모델을 가정한다

- 파일: `frontend/src/App.tsx`
- `channelActionIndex.mayorIsland(slotIndex) = 69 + slotIndex`
- `channelActionIndex.mayorCity(slotIndex) = 81 + slotIndex`
- `confirmMayorDistribution()`는 `mayorPending` 전체 배열을 만든 뒤 island 12칸, city 12칸을 순회하며 slot별 action을 전송한다.
- 파일: `backend/app/services/action_translator.py`
- `mayor_toggle("island", i) -> 69 + i`
- `mayor_toggle("city", i) -> 81 + i`

### 5. 결론: 현재 계약은 서로 다르다

- 엔진/serializer: "현재 커서 슬롯 + 배치 수량"
- 프론트/action_translator: "슬롯 직접 지정 토글"
- 이 상태에서는 Mayor UI가 엔진의 실제 행동 공간을 정확히 표현하지 못한다.

## 수정 설계

### 1. 프론트 Mayor 입력 모델 변경

현재:

```ts
mayorIsland(slotIndex) => 69 + slotIndex
mayorCity(slotIndex) => 81 + slotIndex
```

변경:

```ts
const channelActionIndex = {
  mayorAmount: (amount: number): number => 69 + amount,
};
```

영향:

- 프론트는 더 이상 24칸 전체 계획 배열을 action index로 직접 변환하지 않는다.
- `state.meta.mayor_slot_idx`를 보고 현재 슬롯을 표시하고,
  그 슬롯에 대해 가능한 수량만 `69~72`로 전송해야 한다.

### 2. `confirmMayorDistribution()` 제거 또는 순차 제출 방식으로 축소

현재:

```ts
for (let i = 0; i < 12; i++) {
  for (let j = 0; j < mayorPending[i]; j++) {
    await channelAction(channelActionIndex.mayorIsland(i));
  }
}
```

문제:

- 엔진은 slot index를 받지 않는다.
- city slot 0을 의미하는 `81`은 현재 env step에서 Mayor action으로 소비되지 않는다.

변경 방향:

```ts
const slotIdx = state.meta.mayor_slot_idx;
const amount = plannedAmountForCurrentSlot(slotIdx);
await channelAction(69 + amount);
```

영향:

- 토글형 전체 계획 UI를 유지하려면 별도 변환 레이어가 필요하다.
- 가장 안전한 방식은 전체 계획형 UI를 걷어내고 현재 슬롯 중심 UI로 바꾸는 것이다.

### 3. action translator 문서/함수 정리

현재:

```py
def mayor_toggle(target_type: str, target_index: int) -> int:
    ...
```

변경:

```py
def mayor_place(amount: int) -> int:
    if not (0 <= amount <= 3):
        raise ValueError("Mayor amount out of range")
    return 69 + amount
```

영향:

- 채널/레거시 사이의 Mayor 의미가 하나로 정리된다.
- 잘못된 slot-address 문서가 제거된다.

### 4. 테스트 영향

추가/수정 필요:

- 엔진 characterization test 유지
- serializer contract test 유지
- 프론트 테스트는 `mayor_slot_idx` 기반 렌더/버튼 활성화로 재작성
- `69~80`, `81~92`를 Mayor 슬롯 주소로 설명하는 테스트/주석은 전부 정리 필요
