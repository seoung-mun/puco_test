# PPO Agent Replay Analysis & Training Improvement Plan

## 1. Replay Diagnosis: Is This Agent "Well-Trained"?

### 1.1 Executive Summary

> [!CAUTION]
> **학습 실패로 판단합니다.** 100M step 학습에도 불구하고, 에이전트는 전략적으로 **단일 차원(Shipping VP)에 극도로 편향된 퇴화 정책(Degenerate Policy)**을 학습했습니다. 이는 인간의 직관에 의한 평가가 아니라, Puerto Rico의 게임 메커니즘과 RL 이론 양쪽에서 객관적으로 검증 가능한 실패입니다.

### 1.3 "인간 직관에 의한 평가"인가?

이것은 객관적 실패입니다. 근거:

1. **Game Rule 기반**: Puerto Rico에서 승점의 세 원천(Shipping VP, Building VP, Large Building Bonus)을 종합적으로 활용하는 것이 점수 최대화의 필요조건. 단일 원천에 편향되면 점수 상한이 크게 제한됨
2. **Nash Equilibrium 기반**: 3인 self-play에서 모든 플레이어가 동일 정책이면 Win Rate ≈ 33%이어야 함. 특정 전략이 지배적이면 다른 플레이어가 이를 exploit할 수 있어야 하나, self-play이므로 모두 같은 방식으로 exploit → 결국 아무도 못 함
3. **VP Score 기반**: Winner P0의 38VP 중 32VP(84%)가 Shipping. 숙련 인간 플레이어의 게임에서는 Building VP가 보통 전체의 30~50%를 차지. 이 게임의 Total VP Pool(75칩)에서 무려 **76VP가 소비되어 VP 고갈로 게임 종료** → 과도한 Shipping 루프의 증거

## 2. Root Cause Analysis: 왜 이렇게 학습되었나?

### 2.1 Potential Function의 Shipping VP 편향

현재 `_compute_potential()`:
```python
chip_vp = p.vp_chips           # ← Shipping VP 반영 (즉각적)
building_vps = Σ base_vp       # ← 건물 VP 반영 (구매 시)
dynamic_large_vp = ...          # ← 대형 건물 보너스 (조건부)
total_vp = chip_vp + building_vps + dynamic_large_vp
return total_vp * 0.01
```

**문제점**:
- `chip_vp`(Shipping VP)는 **Captain 역할 때마다 즉각적으로 증가** → PBRS가 즉각적 보상 제공
- `building_vps`는 건물 구매 시에만 증가하지만, 건물 구매에는 **Doubloon 축적 → Builder 선택 → 구매**의 3-step 지연이 있음
- **Mayor를 안 선택해도 Potential이 감소하지 않음** (건물 base VP는 구매 순간 반영됨). 즉 건물을 짓고 Colonist를 배치하지 않아도 패널티가 없음
- **Trader를 활용한 Doubloon 축적의 중간 보상이 0** → Doubloon 자체는 Potential에 반영되지 않으므로, Trading으로 돈을 버는 것에 대한 학습 신호가 없음

### 2.2 Self-Play Relative Overgeneralization

3 self-play 에이전트는 모두 같은 정책을 공유하므로:
- 모든 플레이어가 동일하게 Shipping에 집중 → 서로 견제가 안 됨
- 만약 한 명이 Building Strategy를 쓰면, 다른 두 명이 Captain을 무한 소환하여 VP 칩을 먼저 고갈시킴 → Building 전략이 자연스럽게 불리해 보임
- 결과: **Shipping 전략만 Self-Play 내에서 서로를 강화(mutually reinforcing)**

### 2.3 Entropy Collapse 가능성

- `INITIAL_ENT_COEF = 0.05 → MIN_ENT_COEF = 0.005` (선형 감소)
- 100M step 학습이면 이미 최소값에 도달
- Replay에서 확인 가능: 역할 선택 시 `prob > 0.99`인 경우가 대부분 → **정책이 near-deterministic으로 수렴**

## 3. Proposed Improvement Plan

### Phase 1: Potential Function 근본 개선

> [!IMPORTANT]
> Potential Function이 **건물 가동 상태, Doubloon 가치, 다양한 재화의 잠재적 VP 가치**를 반영하도록 개선해야 합니다.

#### [MODIFY] [pr_env.py](file:///home/daehan/PuertoRico_RL/env/pr_env.py)

**현재 Potential**: `VP_chip + Building_base_VP + Large_building_bonus`

**제안 Potential**:
```python
def _compute_potential(self, player_idx: int) -> float:
    p = self.game.players[player_idx]
    
    # 1. 확정 승점: VP Chips
    chip_vp = p.vp_chips
    
    # 2. 건물 기본 VP (기존과 동일)
    building_vps = sum(BUILDING_DATA[b.building_type][1] 
                       for b in p.city_board 
                       if b.building_type not in (BuildingType.EMPTY, BuildingType.OCCUPIED_SPACE))
    
    # 3. 대형 건물 동적 보너스 (기존과 동일)
    dynamic_large_vp = ... # 기존 로직 유지
    
    # 4. [NEW] 생산 인프라 가치: 가동 중인 생산 시설의 잠재적 VP 기여
    #    Occupied production buildings/plantations → future VP via Craftsman + Captain
    production_capacity = count_active_production_chains(p)
    production_value = production_capacity * 0.5  # 미래 VP의 대리 변수
    
    # 5. [NEW] Doubloon의 잠재적 건물 VP 가치
    #    Doubloons → future Building VP via Builder
    #    게임 진행도에 따라 감소 (후반부에 돈의 가치 하락)
    game_progress = self._game_step_count / self.max_game_steps
    doubloon_value = p.doubloons * 0.3 * (1.0 - game_progress)
    
    # 6. [NEW] Colonist 활용도: 가동률 패널티
    #    Unoccupied buildings/plantations는 자원 낭비
    total_capacity = sum(...)  # 전체 배치 가능 슬롯
    occupied_slots = sum(...)  # 점유된 슬롯
    utilization = occupied_slots / max(total_capacity, 1)
    utilization_value = utilization * 2.0
    
    total_vp = chip_vp + building_vps + dynamic_large_vp 
               + production_value + doubloon_value + utilization_value
    return total_vp * 0.01
```

> [!WARNING]
> **반론**: 이전 대화에서 Human Heuristic을 제거하고 순수 VP 기반 Potential로 갔는데, 다시 Heuristic을 넣는 것이 맞는가? 
>
> **판단**: 현재 순수 VP Potential은 **이론적으로는 sound하지만, 학습 신호가 너무 sparse하여 100M step으로도 효과적 탐색이 불가능**합니다. 실용적으로, Doubloon이나 생산 인프라에 소량의 가중치를 부여하는 것은 "Reward Shaping은 학습 속도 개선이지 최적 정책 변경이 아니다"라는 PBRS 이론과 양립합니다 — 단, Potential이 **현재 state만의 함수**이어야 합니다. 위 제안은 이 조건을 만족합니다.
>
> **대안**: Human Heuristic 추가 없이, Self-Play 구조 자체를 개선하는 것이 더 근본적 해결책일 수 있습니다 (Phase 2 참조).

#### A. Entropy 바닥값 상향

- 학습 초기: Shaping 가중치 높게 → 빠른 기본 전략 학습  
- 학습 후기: Shaping 가중치 0으로 감소 → Terminal Reward만으로 정책 fine-tuning

- 100M step에 Linear Decay는 마지막에 LR이 거의 0 → 정책 개선 불가
- Cosine Annealing 또는 Warm Restart 고려
